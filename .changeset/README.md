# Changesets

This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos to help you version and publish your code.

## Creating a Changeset

When you make changes to a package that should be released, run:

```bash
pnpm changeset
```

This will prompt you to:
1. Select the packages that have changed
2. Choose the type of version bump (major, minor, patch)
3. Write a summary of the changes

## Versioning Strategy

This monorepo uses **fixed versioning** - all `@maison/*` library packages are released together with the same version number.

### Package Categories

**Published Packages** (versioned together):
- **Core**: `@maison/types`, `@maison/utils`, `@maison/config`, `@maison/config-tailwind`
- **API Layer**: `@maison/api-core`, `@maison/api-client`, `@maison/api-medusa`, `@maison/api-bridge`, `@maison/api-sage`
- **State & Hooks**: `@maison/state`, `@maison/hooks`
- **UI**: `@maison/ui-mobile`, `@maison/ui-b2b`

**Private Packages** (apps - not versioned):
- `@maison/web` - Next.js B2B storefront
- `@maison/mobile` - Expo mobile app
- `@maison/backend-medusa` - Medusa V2 backend

### Version Bumps

- **Patch** (0.0.X): Bug fixes, documentation updates, internal refactoring
- **Minor** (0.X.0): New features, backwards-compatible changes, new exports
- **Major** (X.0.0): Breaking changes, API modifications, removed exports

## Commands

```bash
# Create a new changeset
pnpm changeset

# Apply changesets and update package versions
pnpm version

# Check what changesets are pending
pnpm changeset status
```

## Workflow

1. Make changes to packages
2. Run `pnpm changeset` to document changes
3. Commit the changeset file (`.changeset/*.md`)
4. When ready to release, run `pnpm version`
5. Commit the version bumps and updated CHANGELOGs
6. Push and optionally publish to registry

## Example Changeset

```markdown
---
"@maison/api-client": minor
"@maison/types": minor
---

Add support for B2B quote management in api-client with new types
```

Since we use fixed versioning, selecting any package will bump all related packages to the same version.
